# Implementation Plan: Clean Fork for Upstream Sync

**Branch**: `002-clean-runtime-artifacts` | **Date**: 2026-06-22 | **Spec**: [spec.md](file:///d:/Developpement/Projets/WEB/spec-kit/specs/002-clean-runtime-artifacts/spec.md)

**Input**: Feature specification from `specs/002-clean-runtime-artifacts/spec.md`

## Summary

This fork of `github/spec-kit` currently tracks ~295 runtime-generated files
(agent skills, workflows, scripts, templates, integration manifests) that the
upstream project explicitly ignores. These files create merge conflicts on every
upstream sync and provide no shared value — they are regenerated locally by
`specify init`.

The plan removes these files from the git index (without deleting them from
disk), adds `.gitignore` rules to prevent re-tracking, creates a tagged restore
point, and publishes a setup guide so fresh clones can regenerate them. After
this work, upstream merges will produce zero conflicts on runtime paths.

### Phase B: `.gitignore` Update (FR-006)

#### B.1 — Add Fork-Specific Ignore Rules

Append a clearly commented section to `.gitignore`:

```gitignore
# =============================================================================
# Fork-specific: runtime artifacts
# =============================================================================
# These directories are regenerated locally by `specify init` and should not
# be tracked. The upstream github/spec-kit project already ignores them.
# To regenerate after a fresh clone, see docs/fork-setup.md.

# Agent integration directories (skills, workflows, commands)
.agents/
.claude/
.kilocode/
.opencode/

# Generated agent context file (AGENTS.md remains tracked — it is upstream content)
CLAUDE.md

# .specify/ runtime subdirectories (regenerated by specify init)
.specify/scripts/
.specify/templates/
.specify/memory/
.specify/integrations/
.specify/workflows/
.specify/extensions/
.specify/feature.json

# Keep project configuration files tracked
!.specify/extensions.yml
!.specify/init-options.json
!.specify/integration.json
```

**Design decision**: The negation rules (`!`) use the explicit-file pattern
rather than glob patterns. This ensures that only the three known config files
are tracked, and any new runtime files added to `.specify/` are automatically
ignored.

### Phase C: Index Cleanup (FR-001, FR-002, FR-007)

#### C.1 — Remove Agent Integration Directories

```powershell
git rm -r --cached .agents/
git rm -r --cached .claude/
```

This removes 247 files from the index. The `--cached` flag ensures zero local
file deletion (FR-007).

#### C.2 — Remove `.specify/` Runtime Subdirectories

```powershell
git rm -r --cached .specify/scripts/
git rm -r --cached .specify/templates/
git rm -r --cached .specify/memory/
git rm -r --cached .specify/integrations/
git rm -r --cached .specify/workflows/
git rm -r --cached .specify/extensions/
```

This removes 47 files from the index. The three project config files
(`.specify/extensions.yml`, `.specify/init-options.json`,
`.specify/integration.json`) are NOT affected because `git rm --cached`
operates on the specific paths listed, and these files are in `.specify/` root
(not in the subdirectories being removed).

**Note**: `.specify/extensions.yml` sits at the `.specify/` root alongside the
other two config files. The `git rm -r --cached .specify/extensions/` command
targets the `extensions/` *subdirectory*, not the root-level `extensions.yml`
file. These are distinct paths and there is no conflict.

#### C.3 — Remove Generated Context File

```powershell
git rm --cached CLAUDE.md
```

Removes 1 file. `AGENTS.md` is deliberately NOT removed — it is upstream-
tracked project documentation.

## Verification Plan

### Automated Checks (PowerShell)

```powershell
# 1. Runtime paths fully untracked (SC-002)
$runtime = git ls-files .agents/ .claude/ .specify/scripts/ .specify/templates/ `
    .specify/memory/ .specify/integrations/ .specify/workflows/ .specify/extensions/ CLAUDE.md
if ($runtime) { Write-Error "FAIL: Runtime files still tracked: $runtime" }

# 2. Config files still tracked (FR-003)
$configs = git ls-files .specify/extensions.yml .specify/init-options.json .specify/integration.json
if (($configs | Measure-Object -Line).Lines -ne 3) { Write-Error "FAIL: Config files missing" }

# 3. Ponytail still tracked (FR-004)
$ponytail = git ls-files extensions/ponytail/
if (($ponytail | Measure-Object -Line).Lines -ne 6) { Write-Error "FAIL: Ponytail files missing" }

# 4. AGENTS.md still tracked (upstream source)
$agents = git ls-files AGENTS.md
if (-not $agents) { Write-Error "FAIL: AGENTS.md not tracked" }

# 5. Local files exist on disk (FR-007, SC-004)
$paths = @(".agents/skills/api-design/SKILL.md", ".claude/skills/api-design/SKILL.md",
           ".specify/scripts/powershell/common.ps1")
foreach ($p in $paths) {
    if (-not (Test-Path $p)) { Write-Error "FAIL: $p missing from disk" }
}

# 6. Restore point exists (FR-008, SC-005)
$tag = git tag -l "pre-cleanup-002"
if (-not $tag) { Write-Error "FAIL: Restore tag not found" }
```
